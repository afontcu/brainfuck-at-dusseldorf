# Divendres

Functional Javascript (II)

---

què és una funció?

```notes
Takes an input and computes an output and explicitly returns it.
function foo (x) { console.log(x) } // procedure!

A function doesn't "just" takes inputs and compute outputs. it *only* takes
inputs and compute outputs. 

Functions have to take only direct inputs, and compute only direct outputs.

Indirect inputs and outputs are called "side-effects".

The rule is not "No side-effects".
The rule is "Avoid side-effects wherever possible"
```

---

```js
function sum (x, y) {
  return function () {  // <--
    return x + y
  }
}
```

```notes
foo() is lazy. Computating doesn't occur until you execute it:
```

function sum (x, y) {
  var total = x + y  // "work" is done here
  return function () {

  }

```notes

que no hi ha cap reassignament.

"High degree of confidence" que, en aquest context, cridar sum(3,4) dues vegades
donaria el mateix resultat.
```

eager vs. lazy

```notes
Tradeoffs. Potentially unnecessary work vs. Duplicate work.
```

```js
function sum (x, y) {
  var total

  return function () {
    if (total === undefined) {
      total = x + y
    }
    return total
  }
}
```

```notes
(memoization)

is this still a pure function? now we are closed over a variable that absolutely
changes.

here we made a tradeoff.

more performance. less degree of confidence a reader can get.
```
